1.1Java规范

1.1.1文件编码：UTF-8

1.1.2Import语句

不要出现类似这样的import语句：import java.utils.*。import语句不换行

1.1.1.3非空块：K & R 风格

对于非空块和块状结构，大括号遵循Kernighan和Ritchie风格 (Egyptian brackets):

          1）左大括号前不换行

          2）左大括号后换行

          3）右大括号前换行

          4）如果右大括号是一个语句、函数体或类的终止，则右大括号后换行; 否则不换行。

            例如，如果右大括号后面是else或逗号，则不换行。

示例：

 new MyClass() {
   @Override
   public void method() {
       if(condition()) {
          try{
              something();
             } catch (IOException e) {
               recover();
           }
        }
    }

1.1.4 变量声明

不要使用组合声明，比如：int a,b;

1.1.5注释

1）方法名注释 ：/** 方法介绍 */

2）变量名注释： //变量名介绍

3) 代码注释： //代码介绍 或

           /**

            *代码介绍

            **/

1.1.6 静态成员：使用类进行调用

使用类名调用静态成员，而不是具体某个对象或表达式

如:

Foo aFoo = ...;
Foo.aStaticMethod(); // good
aFoo.aStaticMethod(); // bad

1.2Android规范

1.2.1 命名规范

1.2.1.1包（packages）:

采用反域名命名规则，全部使用小写字母。

一级包名为com，二级包名为xx（可以是公司或则个人的随便），三级包名根据应用进行命名，四级包名为模块名或层级名。

如：

          包名                                                    此包中包含
    com.xx.应用名称缩写.activity                              页面用到的Activity类

    com.xx.应用名称缩写.adapter                               页面用到的Adapter类

    com.xx.应用名称缩写.bean(model/data)                      实体类对象

    com.xx.应用名称缩写.view（com.xx.应用名称.ui）            自定义的View类等

    com.xx.应用名称缩写.utils（tools）                        公共工具类

    com.xx.应用名称缩写.service                               Service服务类

    com.xx.应用名称缩写.fragment                              Fragment界面

    com.xx.应用名称缩写.chat                                  聊天相关类

    com.xx.应用名称缩写.json                                  解析工具类

    *com.xx.应用名称缩写.api  

    com.xx.应用名称缩写.dateBase                             数据库相关类

1.2.1.2 类

采用驼峰命名法，尽量避免缩写，除非改缩写是众所周知的，

比如HTML,URL,如果类名称中包含单词缩写，则单词缩写的每个字母均应大写。

如：

              类                       描述                                  例如

         activity 类              Activity为后缀标识                     欢迎页面类WelcomeActivity

         Adapter类                Adapter 为后缀标识                     新闻详情适配器 NewDetailAdapter

        公共方法类                utils或helper为后缀标识                线程池管理类：ThreadPoolUtils

        *数据库类                 以DBHelper（database）后缀标识         新闻数据库：NewDBHelper(NewDatebase)

        Service类                 以Service为后缀标识                    时间服务TimeService

        BroadcastReceive类        以Broadcast为后缀标识                  时间通知TimeBroadcast

       *ContentProvider           以Provider为后缀标识  

       直接写的共享基础类         以Base开头                             BaseActivity,BaseFragment

1.2.1.3 接口

命名规则与类一样采用驼峰命名法，多以Interface结尾。

1.2.1.4 方法

动词或动名词，采用小驼峰命名法，列如： onCreate(),run()

            方法                                           说明

          initXX()                         初始化相关方法,使用init为前缀标识，如初始化布局initView()

 isXX()、checkXX()                         方法返回值为boolean型的请使用is或check为前缀标识

           getXX()                         返回某个值的方法，使用get为前缀标识

       processXX()                         对数据进行处理的方法，尽量使用process为前缀标识

       displayXX() 
            show()                        弹出提示框和提示信息，使用display(show)为前缀标识

          saveXX()                         保存数据相关的，使用sav为e前缀标识

         resetXX()                         对数据重组的，使用reset前缀标识

         clearXX()                         清除数据相关的

       removeXXX()                         清除数据相关的

         drawXXX()                         绘制数据或效果相关的，使用draw前缀标识

1.2.1.5 变量

采用小驼峰命名法。

  控件名称：Button btn_collection;

  boolean类型变量名称：boolean isCollection;

  普通变量名称：String strCollection;

类中控件名称必须与xml局部id保持一致。所有控件名称都以下划线小写字母连接，其余命名名称一律采用小驼峰命名法，布尔类型一is/has开头。

1.2.1.6 常量

全部大写，采用下划线命名法 例如：MIN_WIDTH

1.2.1.7 资源文件

全部小写，采用下划线命名法，加前缀区分命名模式 

                    名称                        功能

                    btn_xx               按钮图片使用btn_整体效果（selector）

                  btn_xx_normal          按钮图片使用btn_正常情况效果

                 btn_xx_press            按钮图片使用btn_点击时候效果

                   bg_head               背景图片使用bg_功能_说明

                  def_search_cell        默认图片使用def_功能_说明

                 icon_more_help          图标图片使用icon_功能说明 或 ic功能_说明

                   selector_             状态（控件）功能名

                   shape_                 圆角（圆弧）方向

                    bg_                   填充色

                   stroke_                线条色

1.2.1.8 资源布局文件（xml文件（layout布局文件））

全部小写，采用下划线命名法

        1) activity布局文件命名： activity_文件名

        2）fragment布局文件命名： frag_文件名

        3）对话框布局文件名：dialog_文件名

        4) Item布局文件：item_文件名

        5) 头部文件： head_文件名

        6）Popwindow文件：pop_文件名

        7）List文件：list_文件名

1.2.1.9 动画文件（anim文件夹下）

全部小写，采用下划线命名法，追加前缀区分。

前面为动画的类型，后面为方向

           动画命名例子          规范写法

                fade_in           淡入   

               fade_out           淡出   

           push_down_in           从下方推入

          push_down_out           从下方推出

              push_left           推像左方    

      slide_in_from_top           从头部滑动进入

                zoom_in           变形进入     

               slide_in           滑动进入

1.2.2.0 资源id

大小写规范与方法名一致，采用小驼峰命名法，命名规范为“资源控件缩写名”+“变量名”。

注意：页面控件名称应该和控件Id名保持一致

1.2.2.1 layout中的布局命名

命名模式为：view缩写_模块名称——view的逻辑名称

View的缩写详情如下：

        SeekBar    skBar

      VideoView    video_

        WebView    web_

      MapView      map_

1.2.3strings.xml文件

strings.xml中使用%1$s实现字符串的通配 除了注释中出现中文，其他地方一律不允许出现

1.2.4 打印信息

使用Log打印，无特殊情况下禁止使用System.out.print();

1.2.5 异常捕捉

尽量捕捉具体的异常，如IOExceptioin等。
